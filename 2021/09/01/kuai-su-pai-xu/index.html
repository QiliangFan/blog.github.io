快速排序

快速排序(quicksort)算法和归并排序算法属于分治策略的典型应用, 但是二者有较大区别.

归并排序的计算量主要消耗在有序子向量的的归并操作上，子向量的划分几乎不花时间。

快速排序可以在$O(1)$ 时间内, 由子问题的解直接得到原问题的解, 但是将原问题划分成两个子问题, 需要$O(n)$时间

  • 快速排序能够确保划分出的子任务彼此独立, 但是却不能保证两个子任务的规模大体相当(甚至极不平衡)

    - 快速排序在最坏情况下不能保证$O(nlogn)$的复杂度

  • 对于随机分布的输入序列，快速排序算法实际的平均运行时间相比同类算法还是要少得多。

1. 轴点 ：$Pivot$

对于一个向量区间 $[lo, hi)$, 如果对于任意的 $lo \le mi \lt hi$, 以元素$s[mi]$为界可以划分出前后两个子向量, $s[lo, mi)$ 和 $s[mi, hi)
$
若$s[lo, mi)$ 中的元素均不大于$s[mi]$, 且$s[mi, hi)$中的元素均不小于$s[mi]
$
则元素$s[mi]$ 称作向量$s$的一个轴点(pivot)

一但轴点左右两个子序列完成了排序，便可在$O(1)$时间内完成排序, 得到整个向量的排序结果

2. 快速排序算法:

快速排序算法可以简单地描述为如下形式: (在实际实现时, hi可作为闭区间, 取length-1, 方便实现)

def quick_sort(num_seq: Sequence, lo: int, hi: int) -> None:
    if hi - lo < 2:
        return
    mi = partition(num_seq, lo, hi)
    quick_sort(num_seq, lo, mi)
    quick_sort(num_seq, mi+1, hi)

由代码可知, 轴点的位置一旦确定, 则只需以轴点为界, 分别递归地对前、后子向量实施快速排序即可。

算法的核心就在于：轴点构造算法partition()如何实现? 可以达到多高的效率?

3. 快速划分算法的一种实现

一般情况下, 一个序列中可能并不存在轴点: 即轴点左右元素的大小关系是混乱的, 左边的元素中有比轴点大的, 而右边的元素中可能有比轴点小的.

为此, 通常我们需要人为地通过交换元素位置，人为地构造出一个轴点 。

def partition(num_seq: Sequence, lo: int, hi: int) -> int:
    # random swap: 相当于随机选取一个pivot
    dst = lo + random.randint() % (hi - lo + 1)
    num_seq[lo] = num_seq[lo] ^ num_seq[dst]
    num_seq[dst] = num_seq[lo] ^ num_seq[dst]
    num_seq[lo] = num_seq[lo] ^ num_seq[dst]
    
    pivot = num_seq[lo]
    
    while lo < hi:
        while lo < hi and pivot <= num_seq[hi]:  # 循环1
            hi -= 1
        num_seq[lo] = num_seq[hi]
        while lo < hi and num_seq[lo] <= pivot:  # 循环2
            lo += 1
        num_seq[hi] = num_seq[lo]
    num_seq[lo] = pivot
    return lo

为了更好的描述快速排序算法，我们私自定义一个状态：游离态<br />

  1. 初始时, pivot所处的元素是游离态
  2. 一个元素将其值赋给游离态的元素后, 其自身成为游离态
  3. 在partion算法中，我们永远只能对游离态元素赋值（因此lo 和 hi要交替取值赋值）
  4. parition算法结束时, num_seq[lo] 一定处于游离态, 且pivot的值要回归到游离态上

对于第四点，num_seq[lo] 一定处于游离态这一点，我们可以模拟出partition算法的状态来描述：

num_seq[hi], 随后num_seq[hi]成为游离态 2.循环1结束的另一个条件是lo>=hi(当元素个数为奇数时不会出现lo>hi的情况, 因为hi每次只移动一步), 此时num_seq[hi]赋值给lo或者hi所处元素 随后针对循环1的结束情况1, 我们再考虑循环2的结束: (因为循环1在第二种情况下结束时, 不会进入循环2) 1.lo < hi时, 要结束循环2, 只能是pivot < num_seq[lo], 随后num_seq[lo]成为游离态, 进入下一个大循环 2.循环2结束的另一个条件是循环1因条件2退出循环或者循环2在循环过程中触发lo>=hi, 两个条件下, 大循环都将最后中止, 此时num_seq[lo]`赋值给lo或者hi所处的元素, 自身成为游离态, 这也是整个大循环最后一次游离态的转移.

由上面的模拟结果可以看出， 无论如何最终大循环结束时, 游离态一定是在lo上的, 因此将pivot的值存在lo所处元素上

4. 复杂度 分析:

考虑这样一种情况，选择的轴点总在序列最左端，且序列已然是有序的，那么完成一次partition算法需要O(n) 的时间复杂度
定义快速排序算法处理规模为n的序列所需的时间记为T(n) ，下面这个递归关系将成立：

$$
T(n)=T(0)+T(n-1)+O(n) = T(n-1)+O(n)
$$

因为递归基的时间复杂度为常数，可以根据地推关系求得：

$$
T(n)=T(n-2)+2\cdot O(n)=\cdots =T(0)+n\cdot O(n) = O(n^2)
$$

也就是说，在最坏的情况下，快速排序算法退化 到和普通排序算法一样的复杂度上。

5. 降低最差情况概率：

在前面的partition算法的实现代码中，可以看到通过将第一个元素与随机一个元素进行交换，达到随机选取pivot的作用，一定程度上降低上述最坏情况出现的概率. 这种方法叫做: 随机法
还有一种方法叫做三者取中法 ：从待排的序列中任取三个元素，将数值居中者作为候选轴点。这种方法比随机选取单个元素的策略或者固定选取某个元素的策略，都能以更小的概率出现最坏情况。

使用随机确定候选轴点 的方式求快速排序算法的平均效率

假设待排序的元素服从独立均匀随机 分布。
partition() 算法经过n-1次比较和最多n+1次移动操作之后，对规模为n的序列的划分结果无非n种可能。
划分所得左侧子序列的长度分别是：0，1，…，n-1。
按照假定情况，每种情况的概率均为1/n，若将算法的平均运行时间记作$\hat{T}(n)$

$$
\hat{T}(n) =(n+1)+(1/n)\times \sum_{k=1}^{n}[\hat{T}(k-1)+\hat{T}(n-k)]
= (n+1) + (2/n) \times \sum_{k=1}^{n}\hat{T}(k-1)
$$

等式两边同时乘以n，则有：

$$
n\cdot\hat{T}(n) = (n+1)\cdot n + 2 \cdot \sum_{k=1}^{n} \hat{T}(k-1)
$$

同理可得：

$$
(n-1)\cdot \hat{T}(n-1) = (n-1)\cdot n + 2\cdot \sum_{k=1}^{n-1}\hat{T}(k-1)
$$

上面两式相减 , 可得:

尽管在极差的情况下，快速排序算法会退化到$O(n^2)$的复杂度，但是在平均情况下，其复杂度仍为$O(nlog_2n)$ 且前面的系数要比其他大多数的算法要小, 常数系数仅为: 1.386 。

6. 应对退化

在第3节 中实现的partition方法此后我们称之为方案A , 在这一节中, 为了考虑退化问题, 我们提出方案B .

方案B与方案A的不同之处在于:

方案A

while lo < hi and pivot <= num_seq[hi]:  # 循环1
while lo < hi and num_seq[lo] <= pivot:  # 循环2

方案B

while lo < hi and pivot < num_seq[hi]:  # 循环1

while lo < hi and num_seq[lo] < pivot:  # 循环2

在方案A 中，遇到重复元素（大小相同的元素）我们是掠过，并继续向前探索，对于方案B ，遇到重复元素，是将其赋给游离态的位置。

在方案B中，这样做的好处在于，遇到重复元素也进行交换，可以在一定程度上降低最差情况出现的概率。但是这样做也会增加更多元素交换操作 ，但并不会影响算法线性复杂度。

由于方案B会反复交换重复元素，这些被交换的元素在原输入序列中的相对位置更难得到保证，快速排序稳定性的不足 更是雪上加霜。
